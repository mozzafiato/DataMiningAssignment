Task 2e¶

DMGI Embeddings¶

In the case of the IMDB for DMGI, we require data which has a structure of the multiplex network and, in addition, node attributes. For this purpose we are using the IMDB-MULTI dataset, which is used by the authors in the original paper of DMGI. It contains 3 labels and 2 relations, in which two movies are connected with a common director ("MDM") or a common actor ("MAM"). The data can be downloaded from: https://www.dropbox.com/s/ntutrhk8nr3vveb/imdb.pkl?dl=0 and should be located in the "\DMGI\data" directory.

    # navigate to deepwalk folder

def load_embeddings(folder):
    rows = []
    i = 0
    index = []
    for file in os.listdir(folder):
        i += 1
        
        
        # load in the embeddings
        # the graphs have different sizes, so we have to average over the nodes
        # so that each graph embedding has the same dimension (1,64)
        row = pd.read_csv(folder + "/" + file,sep=" ",skiprows=1,index_col=0,names=range(64)).mean(axis=0)
        rows.append(row)
        index.append(file[1:file.find(".")])
    X = pd.DataFrame(rows,columns=range(64),index=index).sort_index()
    return X

Results:

  • The NMI is still close to 0, as it was in task 1. The graph shape does not have much to do with the genre, as before.
  • In classification, it gets reasonably good training accuracy, and slightly above random chance for testing accuracy
  • Averaging over the deepwalk embeddings might cause losing a lot of information, which results in the low accuracy

If we compare the two embedding methods, we can see that regarding the clustering task, DMGI obtains a better result over Deepwalk, probably due to the fact it considers more information. In addition, DMGI considers a task with 3 labels, which might be even a more difficult task, yet it performs better. In the case of the classification task, deepwalk has better train accuracy than DMGI, but smaller test accuracy, which might suggests overfitting. It is worth noting that here we haven't tuned the model's hyperparameters for this task, but as for this experiment we can say that both algorithms given better results on a classification task than on a clustering task.

Task 2f¶

We have added 10 additional features, one for each relation, i.e graph (MAM and MDM). The features include the clustering coefficient of the graph and four centrality measures. We have looked at two scenarios: using these features only and using them together with the initial ones.

Note: As only DMGI is compatible for using the node attributes for creating the embeddings, this part is only applicable for the DMGI algorithm.

In [ ]:
additional_features_all = []
for G in Gs:
    clustering = nx.clustering(G)
    deg_centrality = nx.degree_centrality(G)
    close_centrality = nx.closeness_centrality(G)
    bet_centrality = nx.betweenness_centrality(G, normalized = True, endpoints = False)
    pagerank = nx.pagerank(G, alpha=0.8)
    
    node_properties = [clustering, deg_centrality, close_centrality, bet_centrality, pagerank]
    additional_features = np.zeros((n, 5))

    for i in range(3550):
        for j, feature in enumerate(node_properties):
            additional_features[i][j] = feature[i]
    additional_features_all.append(additional_features)
